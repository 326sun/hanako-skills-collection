# Presentation Builder

从大纲或原始材料生成结构化的演示文稿内容。

## 工作流程

1. 用户提供主题/材料 → 确认受众和时长
2. 生成大纲（每张幻灯片的标题和关键信息点）
3. 逐张展开为完整内容
4. 为每张幻灯片生成演讲备注

## 先确认三件事

开始前问用户：
1. **受众**是谁？（领导/同事/客户/公众）→ 决定深度和术语
2. **时长**多少分钟？→ 决定幻灯片数量（约 1.5 分钟/张）
3. **风格**偏好？（数据驱动/故事叙事/问题解决）

## 幻灯片结构模板

### 开场（2-3 张）
1. **标题页**：标题 + 副标题 + 演讲者 + 日期
2. **议程**：3-5 个要点，预览结构
3. **背景/问题**：为什么要关注这个话题

### 主体（N 张，按逻辑推进）
每张幻灯片遵循：
- **一个核心观点**
- 3-5 个支撑要点
- 一个视觉元素建议（图表/示意图/对比表格）

### 结尾（2 张）
1. **总结**：回顾 3 个核心要点
2. **下一步/行动呼吁**：观众应该做什么

## 内容原则

- 每张幻灯片一个核心观点，不是多个观点的堆砌
- 正文用短语，不用完整句（除非是引用）
- 数字 > 形容词（"提升 37%" 不是 "大幅提升"）
- 对比用表格，趋势用折线图，占比用饼图
- 演讲备注包含你真正要说的话，不是重复幻灯片内容

## 输出格式

```markdown
## 演示大纲：[主题]
受众：... | 时长：... 分钟 | 风格：...

### 幻灯片 1：标题
**视觉**：...（建议的图表/图片类型）
- 要点 1
- 要点 2

> 🎤 演讲备注：...

### 幻灯片 2：...
```
